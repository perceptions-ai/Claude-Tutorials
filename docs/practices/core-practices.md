# Claude Code 最佳实践

由社区专家总结的核心使用经验。

## 1. 上下文管理

Claude 拥有长上下文窗口，但无关信息过多会干扰推理。

*   **Do**: 明确指定相关文件路径。
*   **Don't**: 一次性把整个 `node_modules` 扔给它。

## 2. 提示词工程 (Prompt Engineering)

即使是智能如 Claude，好的提示词依然关键。

*   **角色设定**: "你是一个资深的 React 开发者..."
*   **任务拆解**: 将复杂任务拆分为 "首先..."、"然后..."、"最后..."。
*   **示例驱动**: 提供一段你期望的代码风格示例。

## 3. 代码审查 (Code Review)

不要盲目接受 AI 生成的代码。

*   始终运行测试。
*   检查边缘情况。
*   关注安全性漏洞。

## 4. 迭代开发

不要期望一次生成完美代码。采用 "生成 -> 运行 -> 报错 -> 将错误反馈给 Claude -> 修正" 的循环模式极其高效。
